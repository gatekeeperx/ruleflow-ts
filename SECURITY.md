# Security Policy

## Supported Versions

We support security fixes for the latest released version of `@ruleflow-ts/dsl-core`. Older versions may not receive patches.

## Reporting a Vulnerability

- Please do not create a public issue for security-related reports.
- Instead, create a private security advisory via GitHub (Security > Advisories) for the repository `gatekeeperx/ruleflow-ts`, or contact the maintainers through a private channel if available.
- If you cannot open a security advisory, you may open a minimal issue requesting contact for a security report without disclosing details.

## What to Expect

- We will acknowledge your report within 2–5 business days.
- We aim to provide a remediation plan or workaround as soon as possible, and coordinate disclosure timelines when appropriate.
- Please give us a reasonable time window to address the issue before public disclosure.

## Scope

- The primary scope is the `@ruleflow-ts/dsl-core` package published on npm (if/when available).
- Demo apps and private packages are considered out of scope for security fixes but we still welcome reports.

## Responsible Disclosure

We appreciate responsible disclosure. Please avoid sharing exploit details publicly until a fix is available.
